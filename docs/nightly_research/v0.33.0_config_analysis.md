# Analysis of settings.json for Gemini CLI v0.33.0

## Key Changes & New Categories

### 1. Model Configuration & Aliases (`modelConfigs`)
- **Aliases**: You can now define named presets. Built-in ones include `base`, `chat-base`, `fast-ack-helper`, `web-search`, etc.
- **Thinking Config**: Critical for Gemini 3. Supported within `generateContentConfig`.
  - `includeThoughts`: Boolean.
  - `thinkingBudget`: Number of tokens.
  - `thinkingLevel`: "LOW", "MEDIUM", "HIGH" (Gemma 3).

### 2. General Settings (`general`)
- **`defaultApprovalMode`**: `default`, `auto_edit`, `plan`. 
- **`checkpointing.enabled`**: Session recovery.
- **`plan.directory`** & **`plan.modelRouting`**: Integrated planning support.
- **`sessionRetention`**: Auto-cleanup based on `maxAge` or `maxCount`.

### 3. Policy Engine (`policyPaths`)
- `policyPaths`: Array of custom policy files. Replaces the old security model.

### 4. UI Customization (`ui`)
- **`inlineThinkingMode`**: `off` or `full`.
- **`showStatusInTitle`** & **`dynamicWindowTitle`**: Feedback in terminal tab.
- **`customWittyPhrases`**: Personalize the loading experience.
- **`footer`**: Granular control over CWD, context usage, and sandbox status visibility.

### 5. Tools & Sandboxing (`tools`)
- **`disableLLMCorrection`**: (Default true) Prevents the model from "guessing" strings if they don't match.
- **`truncateToolOutputThreshold`**: (Default 40,000) Manages large outputs.
- **`sandbox`**: Boolean or path to profile.

### 6. Experimental Features (`experimental`)
- **`toolOutputMasking`**: Save tokens by hiding intermediate outputs.
- **`enableAgents`**: Crucial for sub-agent support.
- **`jitContext`**: Performance optimization for large repositories.

## Comparison with Current Elite Core (v0.28 era)

| Current Setting (v0.28) | v0.33 Status | Action Required |
| :--- | :--- | :--- |
| `model.name` | Legacy | Move to `modelConfigs` aliases if possible, but still supported in `model`. |
| `tools.allowed` | Active | Keep for trusted tools. |
| `ui.showMemoryUsage` | Active | Keep. |
| `ui.footer.hideContextPercentage` | Active | Keep (Elite Core preference). |
| `experimental.plan` | Active | Now more stable. |

## Strategy for Optimization
1. **Enable Gemini 3 Native Thinking**: Configure `modelConfigs` with `thinkingConfig`.
2. **Optimize Token Usage**: Enable `experimental.toolOutputMasking` and `tools.truncateToolOutputThreshold`.
3. **Enhance Security**: Migrate to the new Policy Engine if needed, but primarily use `defaultApprovalMode: "plan"` for safety.
4. **Agent Efficiency**: Enable `experimental.enableAgents` and configure sub-agent overrides.
